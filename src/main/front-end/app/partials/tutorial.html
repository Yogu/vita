The Preview

1 The Import Panel – the user can set the desired analysis parameters

2 The list of imported documents, correctly imported documents are marked with a checkmark, failed documents with an exclamation mark and still non-completed with the play icon

3 A short preview of the imported/chosen document

4 The user can set the analysis parameters to reanalyze the document

ViTA allows the import of text-, epub2- and epub3-documents. Before starting the import the user can set the analysis parameters. The user can set the count of the Word Cloud items and the granularity of the Graph Network. He can enable the stop word list and the filter for unlikely person and place names. At last he can choose between three NLP tools: ANNIE, OpenNLP and Stanford.

The Preview also provides the user with a list of all documents which have already been imported. If there is a checkmark before the title of the document in the list, the analysis was correctly executed.
If there is an exclamation mark before the title, the analysis has failed. The play icon before the title shows that the analysis of the document is still in progress.
The user has the ability to cancel the analysis of processing documents by clicking on the cancel icon next to the trash icon. He also has the ability to restart the analysis of the canceled documents, by clicking the restart icon next to the trash icon.
On the right a preview of the chosen document is displayed. Again the user has the possibility to restart the analysis, but with new analysis parameters by clicking on Analysis Parameters next to the Rename button below the preview. The reanalyzed document will be added to the list of documents.

The Import

The Import: the purpose of the import is to upload the document and recognize the chapters the document is divided into. The import of text files is especially difficult as there is no standard way of formatting them. Therefore ViTA uses the formatting of the internet portal Project Gutenberg, meaning ViTA adapts the import to the templates of text books which are provided by www.gutenberg.org .

  • Very often books includes meta-data. These are part of the book but not part of the story of the book.

    *** START OF {OPTIONAL: YOU CAN ENTER FOR EXMAPLE THE BOOKNAME HERE} ***
    the end of the story text with:
    *** END OF {OPTIONAL: YOU CAN ENTER FOR EXMAPLE THE BOOKNAME HERE} ***
    the line seperator with:
    *** {OPTIONAL: YOU CAN ENTER FOR EXMAPLE THE BOOKNAME HERE} ***

  • The user can set the meta-Data with:

    The Title - TITLE:
    The Author - AUTHOR:
    The Release Date - RELEASE DATE:
    The Publisher - PUBLISHER:
    The Genre - GENRE:
    The Edition - EDITION:

  • Comments are set with "[" and ends with "]" in the text:

    Here is [the comment] the text

  • Chapters are set with "#" :

    # Chapter 1
    Text of Chapter 1

    # Chapter 2
    Text of Chapter 2

  • Line Breaks uses two "-" :

    Text1
    -
    -
    Text2

The Overview

1 The list of offered interaction possibilities, including the Fingerprint, the Graph Network, the Word Cloud, the Plot View and the profile of a character/place

2 The general information about the document: the name of the author, the year of publication, the main characters, the number of chapters and the total count of words

3 The corresponding pictures to the list of offered interaction

The Overview – a document is imported/chosen and the analysis and interaction can start. This view gives the user some general facts about the document, for example the most frequent occurred characters. Furthermore the Overview gives you links to the various views. Thereby, the user can use the appropriate label in the list of interaction possibilities on the left, or he can select the corresponding picture. Besides the Overview, the tool ViTA offers 7 other views on the chosen document.

The Characters

This view provides the user with a list of characters in the document and their related profile. The most frequently occurring character is already selected. The profile of a character provides the user with the gender, the alternative names, the entities this character occurs most frequently with, and the Fingerprint of this character.
With a click on a co-occurrence, the displayed Fingerprint of the character turns into a combination combination Fingerprint using the already selected character and the chosen co-occurrence.

The Places

Equivalent to the Characters this view provides the user with a list of the occurring places.

The Graph Network

1 Intersection Fingerprint between two characters connected by the selected edge

2 The visualization Graph Network consists of nodes and edges. Nodes represents characters, edges represents the connection between two characters. At the beginning the most important characters are already selected. Clicking on an edge highlights that edge and leads to the Fingerprint of the connected characters being displayed.

3 Reset - deselect all added characters, the most important characters are still selected
Deselect all - deselect all characters

4 Search field – the user can search a character to select him for the Graph Network Network

5 The user can set the range to get the Graph Network for a certain interval of the document

The visualization Graph Network is a representation with nodes and edges of the connection between the characters in the document. Each node represents a character and each edge represents the connection between the two characters. The more important the character, the bigger the node. The more the characters occurs together in a sentence, the wider the edge between the nodes. Right at the start, the most important characters (with the maintained color) are already selected and shown in the visualization. The most common characters with the assigned color are shown in the Graph Network with that same color. The user can add more characters to his selection or deselect characters by clicking on the desired name in the list on the right. If the user crosses an edge between two nodes with the mouse, the edge will be highlighted. If he clicks on it, the Fingerprint between the two characters will be displayed above the Graph Network. With a click on the citation in the Fingerprint, the user is forwarded to the Document. The Graph Network includes another important feature. The user has the ability to set the range for the Graph Network. A user might for example only want to see the Graph Network between chapter 3 and chapter 6.

The Word Cloud

1 The visualization Word Cloud – the more often a word occurs, the larger it is displayed

2 Search field – the user can search a character to select him for the Word Cloud

3 The list of characters, the most common characters are on the top and have their own color

The visualization Word Cloud is a well-known visualization. The Word Cloud is a good visualization to quickly show the most important themes and words of a document. Common words in the document are displayed in a bigger size than less common ones. In this case, the user has the opportunity to see the Word Cloud of the whole document with a click on global in the list on the right. Alternatively he can see the Word Cloud of a single character. This Word Cloud of a character shows words that occur most commonly within the same sentence as that character. The colors of the most common characters are maintained and the characters are displayed with the assigned color in the Word Cloud.

The Plotview

The Plot View is a unique and experimental visualization. The 10 most significant characters are illustrated as lines and every chapter by a node. Lines run from the very left, representing the beginning of the document, to very right, representing the the end of the document. Nodes with a character name next to them show the first occurrence of the respective character. If two or more lines meet in a node, that means that the corresponding characters occurred frequently in the chapter represented by the node. Hovering over a node results in a tooltip being shown which lists the characters and places most commonly occurring at this chapter. Hovering over a line highlights the whole line and the label of the character name belonging to the line.

The Fingerprint

1 The Combination Fingerprint – Fingerprint of multiple characters

2 With a click on Toggle separators the user can show or hide the separators inside the Fingerprint representing the chapters/parts of the document.

The Fingerprint is a visualization that shows the temporal course of characters in a document. The start of the bar represents the beginning and the end of the bar represents the ending of the document. Dark blocks within the Fingerprint represent occurrences of a character in the interval represented by the block. Longer blocks signify repeated occurrences over a longer part of the document. Short blocks appear when a character is only mentioned briefly. In order to have a longer block the character does not need to be mentioned in every consecutive sentence, it is sufficient to be mentioned every few sentences. To select a character the user has to click on the name of the desired character in the list. For deselection the user has to click the same name once again. The user also has the ability to select multiple characters to get the Combination Fingerprint. The Combination Fingerprint is a Fingerprint of the selected characters. If the chosen characters occur together in a sentence, dark blocks will appear just like in the case of the normal Fingerprint.
If the user clicks on one of the blocks, he will be forwarded to the Document. In the Document a citation of the occurrence will be highlighted with a specific color.

The Document

1 The list of chapters the document is divided in

2 The Fingerprint – if the user hovers over a block of the Fingerprint the corresponding text passage will be displayed as a tooltip.

The Document is a representation of the text. ViTA is able to recognize the included chapters of the document. The chapters are shown in the list on the left. The text is displayed on the right.
If the user is forwarded by the Fingerprint, the Graph Network, the Word Cloud or by any other means, the keyword from the other views will then be highlighted in the Document. The Document also includes another important feature. On the right a vertical The Fingerprint is displayed. If the user was forwarded by the Fingerprint, then that same Fingerprint will be vertically displayed here. If a word in the Word Cloud was clicked or a word was searched in the Document, then all occurrences of that word will be displayed in the Fingerprint If the user hovers over a block within the Fingerprint, the corresponding text passage will be displayed as a tooltip.
The User can jump to the next/previous occurrence by clicking on the button with the up/down arrow. He also has the ability to reset the highlighting. Another feature is to search a word or a text passage with the input field on the left in the bottom.
